## PORTFOLIO – CRISTIAN VALLARTA

Este repositorio concentra proyectos de desarrollo de software enfocados principalmente en aplicaciones web dinámicas y plataformas de comercio electrónico. Los proyectos incluidos evidencian experiencia en diseño e implementación de soluciones basadas en arquitectura por capas, gestión de roles de usuario, integración con bases de datos relacionales y estructuración modular mediante herramientas de construcción como Maven.

Entre los proyectos más relevantes se encuentra Mercadito, plataforma tipo marketplace desarrollada con Java y JSP, reconocida con el Segundo Lugar en el 32° Concurso Nacional “Premio a los Mejores Prototipos” del Instituto Politécnico Nacional. El sistema incorpora arquitectura basada en roles (administrador, vendedor y usuario), gestión de catálogo de productos, autenticación y control de sesiones, así como integración con base de datos MySQL.

Asimismo, se incluye ClassOnline, plataforma web educativa orientada a la organización y visualización de contenido académico digital. El proyecto integra recursos multimedia, control de sesiones y una estructura modular bajo estándar Java EE, aplicando principios de desarrollo web dinámico y organización estructurada del código.

Las tecnologías empleadas en los distintos proyectos incluyen:

- Java
- JSP
- HTML
- CSS
- JavaScript
- MySQL
- Maven
- Git

Este portafolio presenta evidencia técnica de mi formación y desarrollo de competencias en ingeniería de software, desarrollo web y gestión de proyectos académicos orientados a soluciones funcionales y estructuradas.

## Estructura del Repositorio

El repositorio se organiza bajo una carpeta principal projects/, donde cada subdirectorio corresponde a un proyecto independiente con su propia estructura interna.

- Mercadito-JSP-Ecommerce/
  Plataforma tipo marketplace con gestión de roles (administrador, vendedor y usuario) e integración con MySQL.

- ClassOnline-JSP/
  Plataforma web educativa para organización de contenido académico y recursos multimedia.

Cada proyecto incluye su propio archivo README con descripción técnica, tecnologías utilizadas y objetivo del desarrollo.
